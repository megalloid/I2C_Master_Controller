# I2C EEPROM Hardware Test — ALINX AX301

## Обзор

Проект для аппаратной верификации I2C Master Controller на плате **ALINX AX301 v2** с FPGA **Cyclone IV EP4CE6F17C8**. По нажатию кнопки выполняется набор тестов записи/чтения в EEPROM **24LC04**, результаты отображаются на 7-сегментном дисплее и светодиодах.

## Плата

| Параметр | Значение |
|----------|----------|
| FPGA | EP4CE6F17C8 (Cyclone IV E, 6K LE) |
| Тактовая частота | 50 МГц |
| EEPROM | 24LC04 (I2C, 512 байт) |
| 7-сегментный дисплей | 6 разрядов, active-low |
| Светодиоды | 4 шт. |
| Кнопки | key1 (запуск), rst_n (сброс) |

## Порядок работы

1. **Включение** → на дисплее `------`, светодиоды погашены
2. **Нажать key1** → запускается серия из 4 тестов
3. **Каждый тест:**
   - Записывает байт в EEPROM по определённому адресу
   - Ожидает 6 мс (цикл записи EEPROM)
   - Читает байт обратно
   - Сравнивает записанное с прочитанным
4. **На дисплее** для каждого теста (500 мс):
   ```
   [N] [P/F] [WW] [RR]
   ```
   - N — номер теста (1–4)
   - P/F — Pass или Fail
   - WW — записанные данные (hex)
   - RR — прочитанные данные (hex)
5. **Светодиоды:** LED[k] загорается если тест k+1 пройден
6. **Итог:** на дисплее `P n   F n` (n — количество)
7. **Повторный запуск:** нажать key1 снова

## Тестовые сценарии

| Тест | Адрес | Запись | Ожидание | LED |
|------|-------|--------|----------|-----|
| 1 | 0x00 | 0xA5 | 0xA5 | LED[0] |
| 2 | 0x01 | 0x5A | 0x5A | LED[1] |
| 3 | 0x10 | 0xFF | 0xFF | LED[2] |
| 4 | 0x11 | 0x00 | 0x00 | LED[3] |

Все тесты работают с блоком 0 EEPROM 24LC04 (адрес устройства 0xA0/0xA1).

## Структура проекта

```
quartus/
├── i2c_test.qpf              # Quartus project file
├── i2c_test_top.qsf           # Pin assignments & device settings
├── i2c_test_top.sdc           # Timing constraints (50 MHz)
├── README.md
└── src/
    ├── i2c_test_top.v         # Top-level: prescaler, core, ctrl, display
    ├── i2c_test_ctrl.v        # Test controller FSM (4 тестов)
    ├── seg_scan.v             # 7-segment multiplexer (6 digits)
    └── ax_debounce.v          # Button debouncer (20 ms)
```

I2C ядро берётся из `../rtl/i2c_master_core.v`.

## Сборка

### Quartus (GUI)

1. Открыть `i2c_test.qpf` в Quartus Prime Lite 22.1+
2. **Processing → Start Compilation**
3. **Tools → Programmer → загрузить .sof на плату**

### Quartus (командная строка)

```bash
cd quartus/
quartus_sh --flow compile i2c_test_top
quartus_pgm -m jtag -o "P;output_files/i2c_test_top.sof"
```

## Параметры I2C

## Возможные проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| Все тесты FAIL | Нет pull-up на SDA/SCL | На плате AX301 pull-up установлены; проверить пайку |
| LED не загораются | EEPROM не отвечает (NACK) | Проверить наличие 24LC04 на плате |
| Дисплей не светится | Неверная полярность сегментов | Проверить версию платы (v1 vs v2) |
| Нестабильные результаты | Помехи на I2C | Уменьшить длину проводов, проверить pull-up |
