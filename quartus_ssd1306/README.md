# SSD1306 OLED Test — Cyclone IV (AX301)

Аппаратный тест I2C Master Controller с OLED дисплеем SSD1306 128×64
на плате ALINX AX301 (EP4CE6F17C8). Вся логика — в железе, без процессора.

## Обзор

Проект демонстрирует два режима отображения, активируемых кнопками:

| Кнопка | Режим | Что показывает |
|--------|------|----------------|
| **KEY2** (`key_start`, M15) | Статический | Текст + фиксированный куб (один кадр) |
| **KEY3** (`key_anim`, M16)  | Динамический | Текст + вращающийся куб (цикл) |

На экране 128×64 рисуется:

```
┌────────────────────────────────────────┐
│              S S D 1 3 0 6             │   ← page 0   (текст, статичный)
│                                        │
│                                        │
│            псевдотрёхмерный            │   ← pages 2..5
│            каркасный кубик             │     (вращается в KEY3)
│                                        │
│                                        │
│             S T A T I C   или          │   ← page 7   (подпись режима)
│               A N I M                  │
└────────────────────────────────────────┘
```

### Последовательность при первом нажатии

1. **Задержка 100 мс** — стабилизация питания SSD1306.
2. **Инициализация** — I2C транзакция: 32 байта команд SSD1306.
3. **Рендер кадра** — scene_renderer формирует кадр во фреймбуфере
   (BRAM 1 KiB):
   - Обнуление 1024 байт.
   - Поворот 8 вершин куба вокруг оси Y (SIN/COS LUT, аппаратные умножители).
   - Псевдо-3D проекция (со сдвигом Y по координате Z).
   - Растеризация 12 рёбер (Брезенхэм, read-modify-write).
   - Blit двух текстовых строк (font ROM 5×7).
4. **Передача** — I2C транзакция: 1 control-байт (0x40) + 1024 пиксельных.
5. *Для KEY3*: угол θ увеличивается, цикл повторяется с шага 3.

### Анимация

В режиме KEY3 угол поворота θ ∈ {0..63} (шаг 2π/64) увеличивается на 1
после каждой передачи кадра. Цикл продолжается до повторного нажатия
KEY3 или нажатия KEY2. Частота обновления ≈ 10 FPS (ограничение I2C 100 кГц
и времени передачи 1025 байт).

## Архитектура

```
ssd1306_test_top
├── ax_debounce ×2        антидребезг KEY2 / KEY3
├── prescaler             делитель 50 МГц → 100 кГц SCL-enable
├── i2c_master_core       побитовый I2C-мастер (из rtl/)
└── ssd1306_ctrl          контроллер сцены + передачи
    ├── scene_renderer    рендер кадра в BRAM
    │   ├── framebuffer   1024 × 8 bit (M9K dual-port)
    │   ├── SIN/COS LUT   4-fold symmetry, 17 записей
    │   ├── vertex rotator  итеративный: 4 такта / вершину × 8
    │   ├── Bresenham rasterizer  по 12 рёбрам, RMW в FB
    │   └── font ROM      5×7, 16 символов
    └── i2c_burst_writer  пакетная запись N байт по I2C (из rtl/)
```

## Подключение SSD1306

Дисплей подключается к шине I2C, общей с EEPROM 24LC04 (разные адреса
⇒ конфликтов нет):

| SSD1306 пин | Плата AX301 | FPGA пин |
|-------------|-------------|----------|
| VCC | 3.3V | — |
| GND | GND | — |
| SDA | I2C SDA | E6 (`i2c_sda`) |
| SCL | I2C SCL | D1 (`i2c_scl`) |

Адрес SSD1306 по умолчанию: **0x3C** (SA0=GND).

## Индикация

### Светодиоды

| LED | Сигнал | Значение |
|-----|--------|----------|
| LED[0] | `busy_o` | Передача кадра идёт |
| LED[1] | `done_latch` | Последний кадр отправлен успешно |
| LED[2] | `err_o` | Ошибка (NACK или потеря арбитража) |
| LED[3] | `animating_o` | Анимация активна |

### 7-сегментный дисплей

6 цифр: цифры 1-3 — HEX-счётчик переданных байт, цифра 0 — индикатор
состояния:

| Символ | Означает |
|--------|----------|
| `-` | Передача в процессе |
| `0` | Успех |
| `E` | Ошибка |
| `A` | Анимация |

## Сборка

### Quartus Prime

```bash
cd quartus_ssd1306
# Открыть ssd1306_test.qpf в Quartus Prime Lite/Standard
# Processing → Start Compilation
# Tools → Programmer → загрузить output_files/ssd1306_test_top.sof
```

### Верификация (verilator lint)

```bash
verilator --lint-only -Wall -Wno-UNUSEDSIGNAL -Wno-PINCONNECTEMPTY \
    -Wno-DECLFILENAME \
    rtl/i2c_master_core.v rtl/i2c_burst_writer.v \
    quartus_ssd1306/src/scene_renderer.v \
    quartus_ssd1306/src/ssd1306_ctrl.v \
    quartus_ssd1306/src/seg_scan.v \
    quartus_ssd1306/src/ax_debounce.v \
    quartus_ssd1306/src/ssd1306_test_top.v \
    --top-module ssd1306_test_top
```
