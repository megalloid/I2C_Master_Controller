# Карта регистров I2C Master Controller

## Обзор

Контроллер предоставляет 7 регистров, доступных через AXI4-Lite интерфейс (32-битная шина данных, адресный шаг 4 байта). Базовый адрес определяется при интеграции в адресное пространство Zynq.

## Таблица регистров

| Смещение | Имя | Доступ | Сброс | Описание |
|----------|-----|--------|-------|----------|
| 0x00 | CTRL | R/W | 0x00 | Управление |
| 0x04 | STATUS | R | — | Статус |
| 0x08 | CMD | W | — | Команда |
| 0x0C | TX_DATA | R/W | 0x00 | Данные для передачи |
| 0x10 | RX_DATA | R | 0x00 | Принятые данные |
| 0x14 | PRESCALE | R/W | 0x00F9 | Делитель частоты SCL |
| 0x18 | ISR | R/W1C | 0x00 | Статус прерываний |

## Описание регистров

### CTRL (0x00) — Управление

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:2 | — | — | 0 | Зарезервированы |
| 1 | IEN | R/W | 0 | Разрешение прерывания (1 = включено) |
| 0 | EN | R/W | 0 | Включение контроллера (1 = включён) |

**Примечания:**
- При `EN=0` контроллер не генерирует SCL/SDA, линии отпущены (high-Z)
- Изменение PRESCALE допускается только при `EN=0`
- При переключении `EN` из 1 в 0 текущая транзакция прерывается

### STATUS (0x04) — Статус

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:4 | — | — | 0 | Зарезервированы |
| 3 | AL | R | 0 | Потеря арбитража (Arbitration Lost) |
| 2 | BUSY | R | 0 | Шина занята (между START и STOP) |
| 1 | RXACK | R | 0 | Последний полученный ACK/NACK (0=ACK, 1=NACK) |
| 0 | TIP | R | 0 | Transfer in Progress (1 = транзакция выполняется) |

### CMD (0x08) — Команда

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:5 | — | — | 0 | Зарезервированы |
| 4 | NACK | W | — | Отправить NACK вместо ACK при чтении |
| 3 | WR | W | — | Записать байт из TX_DATA |
| 2 | RD | W | — | Прочитать байт в RX_DATA |
| 1 | STO | W | — | Генерировать STOP после передачи |
| 0 | STA | W | — | Генерировать START (RESTART если шина busy) |

**Типовые комбинации:**

| CMD | Hex | Описание |
|-----|-----|----------|
| STA + WR | 0x09 | START + запись байта (адрес slave) |
| WR | 0x08 | Запись байта данных |
| WR + STO | 0x0A | Запись байта + STOP |
| RD | 0x04 | Чтение байта (с ACK) |
| RD + NACK + STO | 0x16 | Чтение последнего байта (NACK + STOP) |
| STO | 0x02 | Только STOP |

### TX_DATA (0x0C) — Данные для передачи

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:8 | — | — | 0 | Зарезервированы |
| 7:0 | DATA | R/W | 0x00 | Байт для передачи (включая адрес slave с R/W битом) |

**Формат адресного байта:**

```
TX_DATA = {slave_addr[6:0], R/W}
```

- `R/W = 0` — запись (Master → Slave)
- `R/W = 1` — чтение (Slave → Master)

### RX_DATA (0x10) — Принятые данные

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:8 | — | — | 0 | Зарезервированы |
| 7:0 | DATA | R | 0x00 | Последний принятый байт |

### PRESCALE (0x14) — Делитель частоты

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:16 | — | — | 0 | Зарезервированы |
| 15:0 | PRESCALE | R/W | 0x00F9 | Значение делителя |

**Формула:**

```
SCL = clk / (4 × (PRESCALE + 1))
```

**Расчёт для типовых частот (clk = 100 МГц):**

| Режим I2C | SCL | PRESCALE |
|-----------|-----|----------|
| Standard Mode | 100 кГц | 249 (0x00F9) |
| Fast Mode | 400 кГц | 62 (0x003E) |
| Fast Mode Plus | 1 МГц | 24 (0x0018) |

**Внимание:** изменять PRESCALE только при `EN=0`.

### ISR (0x18) — Статус прерываний

| Биты | Имя | Доступ | Сброс | Описание |
|------|-----|--------|-------|----------|
| 31:2 | — | — | 0 | Зарезервированы |
| 1 | AL_IRQ | R/W1C | 0 | Прерывание потери арбитража |
| 0 | DONE_IRQ | R/W1C | 0 | Прерывание завершения транзакции |

**W1C (Write-1-to-Clear):** запись `1` в бит сбрасывает его. Запись `0` не влияет.

Выход `irq_o` = `IEN & (DONE_IRQ | AL_IRQ)`.

## Пример программирования

### Запись байта 0xA5 по адресу 0x10 в slave 0x50

```c
// 1. Настройка
writel(0x00F9, base + PRESCALE);  // 100 кГц при 100 МГц
writel(0x03,   base + CTRL);     // EN=1, IEN=1

// 2. Адрес slave (write mode)
writel(0xA0,   base + TX_DATA);  // 0x50 << 1 | 0
writel(0x09,   base + CMD);      // STA + WR
while (readl(base + STATUS) & 0x01);  // Ждём TIP=0

// 3. Адрес регистра
writel(0x10,   base + TX_DATA);
writel(0x08,   base + CMD);      // WR
while (readl(base + STATUS) & 0x01);

// 4. Данные + STOP
writel(0xA5,   base + TX_DATA);
writel(0x0A,   base + CMD);      // WR + STO
while (readl(base + STATUS) & 0x01);
```

### Чтение байта из адреса 0x10 slave 0x50

```c
// 1. Адрес slave (write mode) + регистр
writel(0xA0,   base + TX_DATA);
writel(0x09,   base + CMD);      // STA + WR
while (readl(base + STATUS) & 0x01);

writel(0x10,   base + TX_DATA);
writel(0x08,   base + CMD);      // WR
while (readl(base + STATUS) & 0x01);

// 2. Repeated START + адрес slave (read mode)
writel(0xA1,   base + TX_DATA);  // 0x50 << 1 | 1
writel(0x09,   base + CMD);      // STA + WR
while (readl(base + STATUS) & 0x01);

// 3. Чтение + NACK + STOP
writel(0x16,   base + CMD);      // RD + NACK + STO
while (readl(base + STATUS) & 0x01);

uint8_t data = readl(base + RX_DATA) & 0xFF;
```
